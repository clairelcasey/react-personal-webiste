Warbler Flask Application

Key components include:

  • Flask back-end
  • PostgreSQL database through SQLAlchemy
  • Sign-up and authentication using OOP
  • Form validation using WTForms
  • Jinja Templating
  • Robust testing including model and view unit tests

Login/ Logout functionality handled through:

  • User class methods (using OOP)
  • Flask's session object
  • WTForm validation for CSRF protection

Logged in users can:

  • View & follow anyone other user's profile
  • Search for other users
  • Add a message (Warble)

Added "like" functionality includes ability to:

  • Like other user's Warbles
  • Dislike other user's Warbles
  • View likes from anyone's profile page & click to view only likes

Robust testing includes:

  • User model & view functions unit tests
  • Message model & view functions unit tests

Key Takeaways:

  • Necessity of good documentation and organization when working with larger codebases
  • Importance of creating robust tests
  • Improvement using OOP/ an ORM can make with larger codebases
